# C++ 学习笔记：枚举 (Enums)

> **核心概念**：**Enum** 是 "Enumeration" 的缩写，本质上是一个**为整数值赋予名字的集合**。它是提高代码可读性和维护性的重要工具。

## 1. 定义与本质 (Definition)
* **实质**：底层的本质是**整数 (Integer)**。
* **作用**：将一组相关的数值赋予有意义的名称，而不是直接使用“魔术数字”。

## 2. 核心优势 (Advantages)
* **可读性 (Readability)**：`State.RUNNING` 显然比 `1` 更容易理解。
* **逻辑分组**：将相关的状态或选项归纳在一起。
* **类型检查**：在一定程度上限制赋值范围，比纯 `int` 更安全。
* **代码整洁**：减少代码中的硬编码数字。

## 3. 技术特性 (Technical Features)
* **默认值**：如果未指定，第一个元素默认为 **0**。
* **递增规则**：后续元素的值默认在前一个基础上 **+1**。
* **手动指定**：可以为任意元素手动赋值（例如 `A=5, B=10`）。
* **底层类型**：
    * 默认占用 **32位 (4字节)** 空间。
    * 可以显式指定底层类型以节省内存（例如 `: unsigned char`）。
* **限制**：只能使用整数类型，**不能使用浮点数 (float)**。

## 4. 使用限制与冲突 (Limitations)
* **命名空间污染**：普通枚举（非 `enum class`）不是独立的命名空间。
* **命名冲突**：枚举成员的名字在当前作用域全局可见，容易与变量或其他枚举成员重名。
    * *解决方案*：通常使用前缀（如 `COLOR_RED`, `STATE_IDLE`）来避免冲突。

## 5. 常见应用场景 (Use Cases)
* **状态机**：表示游戏或程序的状态（如 `Idle`, `Running`, `Paused`）。
* **日志级别**：区分 Log 重要性（如 `Error`, `Warning`, `Info`）。
* **选项替代**：替代一组相关的 `#define` 宏定义。
