# C++ 类与对象 (Classes & Objects) 学习笔记

> 基于 NotebookLM 思维导图整理

## 1. 面向对象编程 (OOP)
* **地位**：当前最流行的编程范式之一。
* **作用**：一种高效组织代码的风格。
* **核心概念**：**类 (Class)** 与 **对象 (Object)**。

## 2. 类 (Class) 的定义
* **本质**：一种用户**自定义的数据类型**。
* **构成**：将**数据**（属性）与**功能**（行为）组合在一起。
* **语法**：
    * 使用 `class` 关键字定义。
    * **注意**：大括号结束后必须加**分号 (;)**。

## 3. 类成员 (Class Members)
### A. 数据 (变量/Variables)
* 代表对象的**属性**或**特征**。
* *导图示例*：位置坐标 (X, Y)、移动速度 (Speed)。

### B. 功能 (方法/Methods)
* 定义在类内部的**函数**。
* **特点**：
    * 专门用于操作类内部的数据。
    * **优势**：可以简化函数参数（不需要像 C 语言结构体那样把指针传以此来修改数据）。

## 4. 可见性/访问权限 (Access Modifiers)
* **目的**：控制代码安全性，实现封装。
* **Private (私有)**：
    * 类的**默认权限**。
    * 仅类内部可访问，外部不可见。
* **Public (公有)**：
    * 类外部可以访问。
    * 通常作为对外提供的接口。

## 5. 对象 (Object)
* **定义**：类的**实例 (Instance)**（类是图纸，对象是盖好的房子）。
* **使用**：
    * **实例化**：像声明普通变量一样创建对象（如 `Player p1;`）。
    * **访问成员**：使用**点运算符 (`.`)**（如 `p1.move();`）。

## 6. 类的优势
* **代码整洁化**：逻辑更清晰。
* **易于维护**：修改类内部实现不影响外部调用。
* **避免命名冲突**：类定义了独立的作用域。
* **语法糖 (Syntactic Sugar)**：提供了更符合人类直觉的编程方式。
