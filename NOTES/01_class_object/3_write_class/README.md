#  C++ Log 类编写教程学习笔记

> **简介**：这是基于《C++ Log 类编写教程》思维导图整理的学习笔记。本笔记旨在记录如何从零开始构建一个 C++ 日志系统，理解其核心概念、用途及在实际开发中的重要性。

## 1.  核心目标 (Core Goals)

本教程旨在通过编写 Log 类达到以下目的：

* **从零开始编写基础类**：不依赖第三方库，手动实现一个工具类。
* **演示基础阶段概念**：通过代码实践，理解 C++ 面向对象编程（OOP）的基础知识。
* **未来扩展性**：当前仅实现基础功能，为未来引入文件写入、多线程等高级功能打下基础。

## 2.  Log 类的定义 (Definition)

Log 类本质上是一个用于消息管理的工具：

* **管理日志消息**：对程序运行产生的信息进行分类和处理。
* **向控制台打印信息**：主要通过标准输出（stdout/Console）展示关键数据。

## 3.  主要用途 (Main Uses)

在软件开发中，Log 类扮演着多种角色：

* **调试程序**：替代原始的 `cout`，追踪代码执行路径。
* **查看游戏/应用状态**：实时监控程序的运行指标（如帧率、连接状态）。
* **信息转储 (Dump)**：将复杂的数据结构导出为可读文本，便于分析。

## 4.  控制台日志的优势 (Advantages)

为什么我们仍然依赖简单的控制台日志？

* **操作系统内置支持**：无需额外安装环境，利用系统原生 API。
* **比图形渲染更可靠**：当图形界面（GUI）崩溃或未加载时，控制台依然能存活并报错。
* **开发利器**：是程序员在开发过程中最常用、最直接的交互工具。

## 5.  复杂性特征 (Complexity)

日志系统的规模是可变的：

* **可简可繁**：既可以是十几行的简单封装，也可以是庞大的系统。
* **大型系统规模**：在企业级应用中，日志模块可能包含数千行代码。
* **开发价值极高**：无论项目大小，良好的日志系统都能显著提升维护效率。

###  学习重点

1. **类成员变量 (`m_LogLevel`)**：用于在对象内部维护当前的状态（日志过滤级别）。
2. **封装 (`public` vs `private`)**：外部不能直接修改 `m_LogLevel`，必须通过 `SetLevel` 方法，这体现了 OOP 的安全性。
3. **枚举类 (`enum class`)**：比普通 `int` 更安全、更语义化。
